2.6 AI Career Coach (Chat) — Must Use Real Conversation Memory and Real User Context

How to tell it's fake: The coach gives generic advice unrelated to anything in the
user's resume or prior messages; asking a follow-up question ("what about that gap you
mentioned?") produces a response that ignores the earlier turn; the same question asked
twice in a row produces wildly different, contradictory advice (a sign there's no real
grounding — it's just free-floating generation).

What "real" looks like:

1. Every chat session is grounded with real user context at the system-prompt level:
   resume summary, target role, recent career-path/job-match activity — injected
   server-side, not something the user has to repeat themselves
2. Full conversation history (or a real summarized window of it) is sent with every
   request — not just the latest message in isolation
3. The coach can reference specific things from the user's actual data
   ("I see from your resume you have 3 years of React — for the senior role you're
   targeting, let's talk about system design depth")
4. Responses to the same question should be consistent in substance (not necessarily
   word-for-word, but not contradictory) given unchanged context
